Terraform Flask App Deployment

Automated Infrastructure Provisioning for a Flask Application on AWS

Project Overview

This project automates the deployment of a Flask web application on AWS using Terraform. The infrastructure includes:

  • Amazon Elastic Compute Cloud (EC2) instances
  • Amazon Virtual Private Cloud (VPC)
  • Security Groups for access control
  • Elastic Load Balancer (ELB) for traffic distribution
  • Amazon DynamoDB for NoSQL database needs
  • Amazon S3 for storing Terraform state files
  • Amazon CloudWatch for monitoring and logging

Architecture Diagram

Deployment Architecture: AWS ECS, DynamoDB, S3, CloudWatch

Technologies Used

  • Terraform - Infrastructure as Code (IaC) for provisioning AWS resources
  • Flask - Python web framework for building the application
  • AWS ECS - Container orchestration for running the application
  • AWS DynamoDB - Fully managed NoSQL database service
  • AWS S3 - Object storage service for storing Terraform state files
  • AWS CloudWatch - Monitoring and logging service for AWS resources

Deployment Steps

  1. Clone the repository: git clone https://github.com/KobeCyber/TerraformFlaskAppDeploymenmt
  2. Navigate to the project directory: cd TerraformFlaskAppDeploymenmt
  3. Initialize Terraform: terraform init
  4. Review the execution plan: terraform plan
  5. Apply the configuration: terraform apply
  6. Access the Flask application via the public IP of the EC2 instance on port 5000
